◆ compileReadPlan()

Error slmp::highlevel::compileReadPlan ( const std::vector< std::string > & addresses,
ReadPlan & out )

Compile a reusable mixed read plan.

This separates address parsing from repeated polling. The resulting plan can be reused by readNamed or Poller.

Batchable word and dword devices are deduplicated automatically, so requesting D100, D100:S, and D100.3 results in one word-device entry in the compiled plan.

Parameters

outReceives the compiled plan.

◆ parseAddressSpec()

Error slmp::highlevel::parseAddressSpec ( const char * address,
AddressSpec & out )

Parse a high-level address string.

Accepted forms:

  • D100
  • D100:S
  • D200:D
  • D200:L
  • D200:F
  • D50.3
  • M1000

Direct bit devices such as M1000 default to ValueType::Bit. Word devices default to ValueType::U16, except plain LTN, LSTN, and LCN, which default to ValueType::U32 current-value access. Dtype suffixes are interpreted as:

  • :S signed 16-bit
  • :D unsigned 32-bit
  • :L signed 32-bit
  • :F float32

For addresses without an explicit suffix, AddressSpec::explicit_type is false, which allows higher layers to preserve the user's original intent.

Parameters
addressAddress string to parse.
outParsed result.
Returns
Error::Ok on success, otherwise an error describing why the address cannot be interpreted by the helper layer.

◆ readTyped() [2/2]

Error slmp::highlevel::readTyped ( SlmpClient & client,
const char * device,
const char * dtype,
Value & out )

Read one logical value by device string and explicit dtype.

Supported dtypes: BIT, U, S, D, L, F.

D, L, and F consume two consecutive PLC words in little-endian order. BIT is intended for direct bit devices such as M1000; to access a bit inside a word device use the address-form overload with .bit notation.

Example:

slmp::highlevel::Value value;
slmp::highlevel::readTyped(plc, "D100", "U", value);
