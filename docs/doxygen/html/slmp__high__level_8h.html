Detailed Description

Optional high-level helper layer for the minimal SLMP client.

This header adds a user-facing facade similar to the Python and .NET helper APIs while keeping the core transport and codec in slmp_minimal.h unchanged. The high-level layer accepts string device addresses, converts them into typed values, and can compile mixed snapshot plans for repeated reads.

Design intent:

  • keep slmp_minimal.h deterministic and buffer-oriented
  • add convenience only in this optional layer
  • allow applications to start with string addresses and typed values
  • keep it easy to drop back to the core API when tighter control is needed
